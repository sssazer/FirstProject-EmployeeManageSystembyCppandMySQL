# 职工管理系统

## 基本介绍

基于控制台创建的职工管理系统，主要使用语言为C++，数据存放在MySQL数据库中

- 使用方式：

下载这个文件夹之后，直接点击.sln即可打开项目工程（需要VS2019或更高的版本）

- 注意事项：

  - 要先将名为manage_system_db的数据库创建好。（表结构见source_code分支中的SqlCode文件，或者直接使用Database File中的manage_system_db数据库文件）
  
  - 在三个类的构造函数中都有登录MySQL数据库的操作，要记得把账号密码改成自己的
  
- 功能说明：
  
  - 进入系统后首先要进行登录，用户的账号密码在数据库中存储，用户分为普通用户和root用户
  
  - 使用普通用户只能登录职工信息管理系统，只能对职工信息进行查看操作
  
  - 使用root用户登录之后，选择要进入的系统：用户账号管理系统和职工信息管理系统
  
    用户账号管理系统：实现对用户账号的增删改查操作
    
    职工信息管理系统：实现对职工信息的增删改查操作

## 0.00（原始版本）

相当于只是为数据库添加了一层接口，所有的操作都只是在C++程序中进行简单的字符串处理，没有任何检测工作，只是拼接成SQL语句之后直接丢给SQL处理。

程序中没有使用任何数据结构对数据进行保存，定义的几个类只是操作类，只是机械地套用类的语法，没有体现任何面向对象的思想，

## 3.12

**界面**

优化界面

**数据存储**

- 用户账号信息

  定义一个基类Account，下面继承两个类：管理员（Admin）和普通用户（User）。用于管理所有用户账号。但是在底层数据库中依然将所有账户存储在一张表中，用权限字段加以区分

  其中存储了比如：账号、密码、上次登录操作的时间（登录成功或失败都算一次操作）

- 员工信息

  定义一个员工类，包含员工所有基本信息

**核心逻辑**

- 每次打开系统时，为管理员和普通用户分别创建一张哈希表，key=账号，value=用户账号对象（其中存储了账号的详细信息）

  将验证账户的逻辑全部移至管理系统中，不再去每次查询数据库，提高运行速度

- 新增一个MySQL助手类，用于封装对SQL的操作，将SQL语句拼接等操作移至助手类中实现。
- 新增员工或账号时，先将信息写入一个对应的对象中，在前端完成对数据的检测的工作。之后再写入数据库中
